## Описание проекта

Этот проект представляет собой реализацию поискового движка для корпоративного портала компании "АйТиБокс".  Цель проекта – разработать систему, аналогичную поисковой строке Яндекса или Google, способную строить поисковые индексы, работать с конфигурационными файлами и предоставлять релевантные результаты поиска.

## Стек используемых технологий

*   **Язык программирования:** C++
*   **Среда разработки:** CLion
*   **Система сборки:** CMake
*   **Библиотека для юнит-тестирования:** Google Test (GTest)
*   **Библиотека для работы с JSON:** nlohmann_json
*   **Операционная система:** Windows

## Краткая инструкция по запуску проекта локально

### Предварительные требования

1.  Установленный компилятор C++ (например, MinGW для Windows, GCC для Linux, Clang для macOS).
2.  Установленная IDE CLion или другая среда разработки, поддерживающая CMake.
3.  Установленная библиотека nlohmann_json.

### Шаги запуска

1.  Клонируйте репозиторий:
        git clone https://github.com/Dima247383/MyProject.git
    cd MyProject
    

2.  Создайте директорию для сборки:
        mkdir build
    cd build
    

3.  Сгенерируйте файлы сборки CMake:
        cmake ..
    

4.  Соберите проект:
        make
    
    Или, если вы используете Windows и MinGW:
        mingw32-make
    

5.  Запустите приложение:
        `MyProject.exe` для Windows

### Настройка приложения

Приложение конфигурируется через файлы формата JSON, расположенные в директории с проектом:

*   **`config.json`:** Содержит общую информацию о поисковом движке, список файлов для индексации и другие параметры.
*   **`requests.json`:** Содержит поисковые запросы.
*   **`answers.json`:** Файл, в который приложение записывает результаты поиска. Если файл отсутствует, он будет создан. Существующий файл будет очищен перед записью.

#### Пример файла `config.json`:
{
  "config": {
    "name": "SkillboxSearchEngine",
    "version": "0.1",
    "max_responses": 5
  },
  "files": [
    "../resources/file001.txt",
    "../resources/file002.txt",
    "../resources/file003.txt",
    "../resources/file004.txt"
  ]
}
#### Пример файла `requests.json`:
{
  "requests": [
    "some words..",
    "some words..",
    "some words..",
    "some words.."
  ]
}
